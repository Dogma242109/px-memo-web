# ロードマップ

## フェーズ0: 設計

- プロジェクト概要を作る
- 要件定義を作る
- 初期アーキテクチャを決める
- データ設計の候補を出す
- 画面設計を作る
- レビュー運用を決める

## フェーズ1: 最小プロトタイプ

- HTML/CSS/JavaScriptの基本構成を作る
- Canvasに線を描けるようにする
- ペン色を変更できるようにする
- ペンサイズを変更できるようにする
- 6色固定パレットを使えるようにする
- 表示倍率を変更できるようにする
- Speed 0-10の再生速度プリセットを使えるようにする
- 現在のフレームを白紙に戻せるようにする
- Undo/Redoできるようにする
- フレームを追加できるようにする
- フレームを切り替えられるようにする
- 再生できるようにする

## フェーズ2: ローカルアプリとして整える

- フレーム削除
- フレーム複製
- 方眼背景
- オニオンスキン
- ズーム編集
- localStorageまたはIndexedDB保存
- JSON書き出し
- JSON読み込み

## フェーズ3: 書き出し機能

- PNG書き出し
- PNG連番書き出し
- GIF書き出し
- WebM書き出し

## フェーズ4: バックエンド学習

- FastAPI導入
- 保存API作成
- SQLite導入
- 作品一覧API
- 作品詳細API
- 作品更新API
- 作品削除API

## フェーズ5: アカウントと同期

- ユーザー登録
- ログイン
- パスワードハッシュ化
- ユーザーごとの作品保存
- クラウド保存
- 複数端末同期

## フェーズ6: リアルタイム機能

- WebSocket導入
- お絵描きチャット
- リアルタイム描画共有
- ルーム機能
