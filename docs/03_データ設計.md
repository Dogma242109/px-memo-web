# データ設計

## 初期の作品データ

初期段階では、作品データをJavaScriptオブジェクトとして扱い、必要に応じてJSONとして保存する。

```json
{
  "id": "project-id",
  "title": "Untitled",
  "width": 640,
  "height": 480,
  "fps": 8,
  "frames": []
}
```

## フレームデータ候補

フレームデータの保存方法は、初期実装で検証する。

### 案A: Image Data URL方式

```json
{
  "id": "frame-id",
  "image": "data:image/png;base64,..."
}
```

良い点:

- 実装がわかりやすい
- Canvasから取得しやすい
- JSONとして扱いやすい

気になる点:

- データサイズが大きくなりやすい
- 差分管理には向かない

### 案B: 描画命令方式

```json
{
  "id": "frame-id",
  "strokes": [
    {
      "color": "#000000",
      "size": 4,
      "points": []
    }
  ]
}
```

良い点:

- あとから編集しやすい
- データサイズを抑えられる可能性がある
- Undo/Redoと相性が良い

気になる点:

- 実装がやや複雑
- 再描画処理が必要

## 初期判断

最初は実装を簡単にするため、Image Data URL方式を優先候補とする。

ただし、Undo/Redoや編集性を重視する段階で、Drawing Commands方式への移行を検討する。
